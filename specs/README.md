# Specs

Escreva um spec **antes** de iniciar qualquer tarefa. O spec é o contrato entre você e o agente.

## Fluxo
1. Crie `specs/minha-tarefa.md` usando `template.md`
2. Inicie a sessão: "Leia specs/minha-tarefa.md e implemente seguindo o spec"
3. O agente explora, planeja, executa e verifica contra o spec
4. Confira o resultado. Se passar nos critérios, está pronto.

## Regras
- Um arquivo por tarefa
- Máximo ~400 palavras (se precisar de mais, divida em subtarefas)
- Critérios de verificação DEVEM ser testáveis
